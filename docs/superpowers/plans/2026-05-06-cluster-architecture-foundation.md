# Cluster Architecture Foundation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the cluster architecture infrastructure (registry, components, integration, llms.txt, schema) that turns 35+ existing pages into a topic-cluster topology with hub-and-spoke linking, and lay tracking groundwork for SEO measurement.

**Architecture:** Single cluster registry in TypeScript (`src/lib/clusterMap.ts`) is source of truth for hub→spokes, spoke→hub, and sibling relationships. Three new Astro components (`HubSpokeList`, `RelatedGuides`, `ServiceCTA`) read from the registry and render structural internal links across service hub pages and academy articles. GEO layer adds llms.txt manifest, HowTo schema for tutorials, and academy-mode FAQ extension. Tracking infrastructure (`ai_referral` event, dashboards, citation log) is set up upfront for baseline measurement.

**Tech Stack:** Astro 4, TypeScript (strict), TailwindCSS 3, JSON-driven i18n (no Astro Content Collections). No test framework — validation is `npm run check` + `npm run build` + dev server visual inspection + Google Rich Results Test for schema.

**Scope:**
- ✅ Phase 0 (Foundation & Tracking): tracking docs, ai_referral event, datasheet skeletons
- ✅ Phase 1 (Architecture refactor): cluster registry, 3 components, page integration, content reclassification
- ✅ Phase 4 (GEO Layer code): llms.txt generator, HowTo schema component, FAQ academy-mode

**Out of scope (separate plans):**
- ❌ Plan 2: Programmatic area×service pages (Phase 2)
- ❌ Plan 3: Content writing (Phase 3 — 60 articles + service hub citation-ready rewrites + academy FAQ buildout)
- ❌ Phase 5 owner-only off-site work (no plan needed)
- ❌ Phase 6 monthly iteration (operational, not implementation)

**Key context (from spec + codebase exploration):**
- Active service hubs are **6 of 9** — trim-restoration, wheel-ceramic, engine-bay are hidden (no `.astro` pages, no entries in `pricing` page). The cluster registry includes all 9 in JSON for forward compatibility, but components only render against active hubs.
- Vertical hubs (4): `/business`, `/ev`, `/plans`, `/commercial-glass`.
- Academy slugs use hyphens (`paint-correction-cost-barcelona`); content keys use underscores (`paint_correction_cost_barcelona`). Mapping: `slug.replace(/-/g, '_')`.
- Articles are filtered by `status === 'published'` in `[slug].astro`; we use `status: 'hidden'` to hide motorcycle.
- Tracking is set up in `src/layouts/BaseLayout.astro` via `window.__rl_push()` global (GTM + Clarity + Meta Pixel). New events plug into this.
- All user-facing text lives in `src/content/{en,es,ca}.json` — 3 files always change together for any text addition.

```js
      // ── AI Referral detection ────────────────────────────────────
      // On page load, check document.referrer against known AI search
      // referrers and push an `ai_referral` event to the unified pipeline.
      (function detectAIReferral() {
        try {
          const ref = document.referrer || '';
          if (!ref) return;
          const aiHosts = [
            { host: 'chat.openai.com', source: 'chatgpt' },
            { host: 'chatgpt.com', source: 'chatgpt' },
            { host: 'perplexity.ai', source: 'perplexity' },
            { host: 'www.perplexity.ai', source: 'perplexity' },
            { host: 'claude.ai', source: 'claude' },
            { host: 'copilot.microsoft.com', source: 'bing_copilot' },
            { host: 'bing.com', source: 'bing' }, // includes bing.com/chat
            { host: 'gemini.google.com', source: 'gemini' },
          ];
          const url = new URL(ref);
          const match = aiHosts.find(({ host }) => url.hostname === host || url.hostname.endsWith('.' + host));
          if (!match) return;
          if (typeof window.__rl_push === 'function') {
            window.__rl_push({
              event: 'ai_referral',
              ai_source: match.source,
              ai_referrer_url: ref,
              landing_path: location.pathname,
              page_lang: pageLang,
            });
          }
        } catch (e) {
          // referrer parse failure — silent fail
        }
      })();
```

```ts
// ── Reverse index helpers ──────────────────────────────────────

/**
 * Given an academy article slug, return the cluster it belongs to.
 * Returns null if the slug isn't in any primary or sub-cluster spoke list.
 */
export function findClusterForSpoke(spokeSlug: string): { clusterId: string; cluster: ClusterDef; isSubCluster: boolean } | null {
  for (const [clusterId, cluster] of Object.entries(clusterMap)) {
    if (cluster.primarySpokes.includes(spokeSlug)) {
      return { clusterId, cluster, isSubCluster: false };
    }
    if (cluster.subClusterSpokes?.includes(spokeSlug)) {
      return { clusterId, cluster, isSubCluster: true };
    }
  }
  return null;
}

/**
 * Given a hub ID, return up to N sibling spokes for a given spoke slug.
 * Excludes the spoke itself. Returns spokes from the same cluster (primary first, then sub).
 */
export function getSiblingSpokes(spokeSlug: string, limit = 3): string[] {
  const found = findClusterForSpoke(spokeSlug);
  if (!found) return [];
  const all = [...found.cluster.primarySpokes, ...(found.cluster.subClusterSpokes ?? [])];
  return all.filter((s) => s !== spokeSlug).slice(0, limit);
}

/**
 * Return only the ACTIVE hubs (hubActive: true).
 * Used by HubSpokeList and area page dense links — we don't link to dead hubs.
 */
export function getActiveClusterIds(): string[] {
  return Object.entries(clusterMap)
    .filter(([_, c]) => c.hubActive)
    .map(([id]) => id);
}
```

```js
      englishLabel: labelMatch?.[1] ?? id,
      englishBrief: briefMatch?.[1] ?? '',
      startingPriceEur: priceMatch?.[1] === 'null' ? null : Number(priceMatch?.[1] ?? 0),
      active: activeMatch?.[1] === 'true',
    };
  }
  return out;
}

async function loadAreaProfiles() {
  const txt = await fs.readFile(path.join(ROOT, 'src/lib/areaProfiles.ts'), 'utf8');
  const out = [];
  const blockRe = /'([\w-]+)':\s*\{([^}]+)\}/gs;
  let m;
  while ((m = blockRe.exec(txt))) {
    const slug = m[1];
    const body = m[2];
    const nameMatch = /name:\s*'([^']+)'/.exec(body);
    if (nameMatch) out.push({ slug, name: nameMatch[1] });
  }
  return out;
}

function pricingPackages() {
  // Static — pricing tiers are defined in content JSON pricing section but
  // we keep canonical EUR figures here for AI clarity. Keep in sync with pricing page.
  return [
    { name: 'Express Refresh', price: 149 },

    { name: 'Two-Stage Correction', price: 549 },
    { name: 'Ceramic Coating 2Y', price: 649 },
    { name: 'Ceramic Coating 5Y', price: 1049 },
  ];
}
```
